# Before Starting Work

Every AI must verify:

- git status  
- git log --oneline -12  
- Branch  
- Current Release  
- Current Phase  
- Current Task  

If not confirmed, stop.

---

# During Work

Do not invent business logic.  
Do not change architecture without approval.  
Do not modify approved Baseline documents casually.  
Keep commit focused.  

One task.  
One commit.  
One purpose.

---

# Before Commit

Verify:

- Working Tree Clean  
- Expected File Only  
- Correct Commit Message  
- Correct Document ID  
- Correct Path  
- No unexpected files  

---

# After Commit

Always report:

- Created File  
- Commit Hash  
- Commit Message  
- Git Status  
- Recent Commits  
- Push Result  
- Current Phase Progress

# When AI Should Stop

AI must stop immediately if:

- Business requirement is unclear.  
- Architecture conflicts.  
- Git is not clean.  
- Multiple writing processes exist.  
- Repository status is uncertain.  
- Founder instruction conflicts.  

Never continue by guessing.
