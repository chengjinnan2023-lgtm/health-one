## 4. Where Staff Hesitated

### 最卡步骤：S4 服务记录 — 保存动作

**现象：** 店员在 S4 填写完服务类型和服务内容后，在"保存服务记录"按钮前犹豫。不是系统报错——按钮可以点击，API 正常返回——但店员不确定是否应该点这个按钮。

**根因分析：**
- "保存服务记录"这个按钮文案暗示"只是保存"，而不是"完成并继续"
- 店员不确定保存后会跳转到哪里
- 按钮位置在页面底部，与"返回"按钮并列，缺少主次区分
- 没有"保存后会自动进入下一步"的提示

**严重级别：P1** — 不阻塞功能，但增加操作犹豫时间。

### 第二卡点：S5→S6 流转不明确

**现象：** S5 反馈保存成功后，屏幕显示"✓ 反馈已记录"，下方有两个按钮："返回客户总览"和"创建随访 →"。但店员第一反应是看"返回客户总览"——因为它是第一个按钮，颜色更熟悉（蓝色 vs 靛蓝）。

**根因分析：**
- 成功页的两个按钮优先级不明确
- "创建随访 →"使用靛蓝色（indigo），与全站蓝色（blue）不一致
- 缺少明确引导语："下一步：为这位客户创建随访"
- 如果店员点了"返回总览"，就漏掉了 S6

**严重级别：P1** — 直接导致 S6 被漏掉。

---

## 5. Most Common Missed Step

### S6 随访 — 最容易漏掉

**现象：** 店员在 S5 成功后倾向于点"返回客户总览"而不是"创建随访 →"。导致客户的随访任务没有被创建。

**根因分析：**
1. S5→S6 流转依赖店员主动点击特定按钮（被 S5 成功页的设计弱化）
2. 没有强制流转——店员的自然反应是"做完了，回去"
3. "随访"概念对店员不够直观——店员认为"服务结束 = 工作结束"
4. 随访不是即时发生的——店员的心理模型是"现在做的事=今天的事"

**严重级别：P1** — 闭环的关键步骤被系统性跳过。
