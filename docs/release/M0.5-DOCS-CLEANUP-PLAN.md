# M0.5 Docs Cleanup Plan
# M0.5 文档整理计划

## Purpose

在进入 M1 Architecture Freeze 之前，统一 Health One 文档入口，避免根目录文档与 docs/ 编号文档长期分叉。

## Current Issue

当前根目录存在：

- `AI_START_HERE.md`
- `PROJECT-MAP.md`
- `PROJECT-MEMORY.md`
- `PROJECT-CONTEXT.md`
- `Glossary.md`

同时 `docs/` 下已有编号版本：

- `docs/02-AI_START_HERE.md`
- `docs/03-PROJECT_MAP.md`
- `docs/04-PROJECT_MEMORY.md`
- `docs/05-PROJECT_CONTEXT.md`
- `docs/06-GLOSSARY.md`

存在长期不同步风险。

## Decision

`docs/` 编号文件作为 **Canonical Source**。

根目录保留轻量 **Redirect** 文件，不再维护完整内容。

## Cleanup Scope

M0.5 只做文档整理，不做架构设计，不写业务代码。

## Tasks

1. 将 `docs/` 编号文件确认为正式版本。
2. 将根目录重复文档改为 Redirect。
3. 保留 `README.md` 和 `PROJECT.md` 作为入口文件。
4. 更新 `README.md` 文档导航。
5. 检查所有引用路径。
6. 完成后进行 M0 Foundation Final Commit。

## Non-Goals

- 不修改业务代码
- 不操作 Legacy 工程
- 不设计 RFC
- 不创建数据库
- 不迁移旧系统代码

## Completion Criteria

完成后，仓库内只有一套正式项目事实来源：

```
docs/00-CONSTITUTION.md
docs/01-GOVERNANCE.md
docs/02-AI_START_HERE.md
docs/03-PROJECT_MAP.md
docs/04-PROJECT_MEMORY.md
docs/05-PROJECT_CONTEXT.md
docs/06-GLOSSARY.md
```
