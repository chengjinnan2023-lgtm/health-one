## 6. Rollback Plan

### If Pilot Fails

```
1. Stop systemd service
   sudo systemctl stop health-one-platform

2. Restore database from backup
   psql health_one_platform < platform-YYYYMMDD-HHMMSS.sql
   cp store-001-YYYYMMDD-HHMMSS.db data/store-001/store.db

3. Revert to previous git tag (if code change caused failure)
   git checkout v0.3.0

4. Restart service
   sudo systemctl start health-one-platform

5. Verify health
   curl https://store.example.com/health
```

### Rollback Triggers

- Data loss detected (Timeline entries missing)
- Staff unable to complete manual loop (system regression)
- Auth bypass discovered (security regression)
- Customer data visible across stores (isolation failure)
