## 2. P1 Usability Issues — Root Cause & Fix Strategy

### P1-1: S4 保存按钮让人犹豫

**店员心理模型：**
> 我填完了服务信息 → 看到"保存服务记录"按钮 → 这是"保存草稿"还是"提交并继续"？→ 不确定 → 犹豫

**根因：** 按钮文案暗示"保存"而非"提交并继续"。店员不知道保存后会发生什么。

**修复策略：** 改按钮文案 + 增加一步流转提示，不改后端逻辑。

### P1-2: S5→S6 流转引导不足

**店员心理模型：**
> 反馈保存成功 → 看到两个按钮 → "返回客户总览"是蓝色的（熟悉）→ "创建随访 →"是靛蓝色的（不熟悉）→ 点返回

**根因：** 成功页的设计优先级反了——主路径（继续到 S6）视觉权重低于次路径（返回 S2）。

**修复策略：** 交换按钮优先级——主路径用蓝色主按钮，次路径用灰色文字链接。

### P1-3: S6 随访容易漏掉

**店员心理模型：**
> 服务做完了，反馈填完了 → 工作结束了 → 回到总览

**根因：** 店员不认为"创建随访"是本次服务的必要步骤。随访在心理模型中是"以后的事"。

**修复策略：** 减少 S5→S6 的选择成本——自动进入 S6，在 S6 内提供"跳过"。

## 3. What to Change Before Pilot #2

### Change 1: S4 保存按钮 + 流转提示

**文件：** `ServiceRecordScreen.tsx`

| 元素 | Before | After |
|------|--------|-------|
| 保存按钮文案 | "保存服务记录" | **"保存并继续 →"** |
| 按钮下方提示 | "必填：服务类型 + 内容 ≤ 5 项" | **"保存后自动进入服务反馈"** |

**尺寸：** 1 行文案修改 + 1 行提示修改。5 分钟。

### Change 2: S5 成功页按钮优先级

**文件：** `FeedbackRecordScreen.tsx`

| 元素 | Before | After |
|------|--------|-------|
| 主按钮（创建随访） | `bg-indigo-600` 靛蓝色 | **`bg-blue-600` 蓝色（与全站一致）** |
| 主按钮文案 | "创建随访 →" | **"下一步：创建随访 →"** |
| 次按钮（返回总览） | `bg-blue-600` 蓝色按钮 | **灰色文字链接 "稍后创建随访"** |
| 成功页引导语 | 无 | **增加 `<p>` "服务反馈已保存。接下来为这位客户创建随访计划。"** |

**尺寸：** 按钮颜色 + 文案 + 1 行引导语。5 分钟。

### Change 3: S5→S6 自动跳转

**文件：** `FeedbackRecordScreen.tsx`

| 元素 | Before | After |
|------|--------|-------|
| 保存成功后 | 显示成功页，两个按钮 | **1.5 秒后自动跳转到 S6** |
| 跳转行为 | 无 | **`setTimeout(() => navigate(...), 1500)`** |

**尺寸：** 1 行 `setTimeout`。2 分钟。

**注意：** 自动跳转后 S6 仍有"跳过"按钮，店员可以随时返回 S2。自动跳转降低选择成本——想继续就什么都不用点，想跳过就点跳过。

### Change 4: S6 页面引导语

**文件：** `FollowUpScreen.tsx`

| 元素 | Before | After |
|------|--------|-------|
| 页面副标题 | "确保服务不因客户离店而结束。" | 不变（已足够好） |
| "跳过"按钮 | "跳过" | **"跳过（稍后补录）"** |
| 表单底部提示 | 无 | **增加 "创建随访后可在客户总览页查看和标记完成"** |

**尺寸：** 2 个文案修改。2 分钟。

## 4. What NOT to Change

### 不要通过改代码解决的问题

| 问题 | 为什么不能通过代码解决 | 正确解决方式 |
|------|----------------------|------------|
| 店员不理解"随访"概念 | 概念本身需要通过培训建立 | 开场 briefing 增加一句解释 |
| 店员以为 S5 就结束了 | 任何 UI 引导都无法替代"这个步骤很重要"的口头强调 | 带教员在 S5 保存后口头提醒："还有最后一步——创建随访" |
| 店员打字慢 | 不是 UI 问题 | 减少必填项已在 Sprint-2 完成；语音输入是 Sprint-4+ 的事 |

### 不要做的"优化"

| 不做 | 原因 |
|------|------|
| S6 改为强制步骤（不允许跳过） | 有些场景确实不需要随访——比如客户明确表示不需要 |
| S4/S5/S6 合并为一个长页面 | 增加单页复杂度，店员更困惑 |
| 增加"教程/引导浮层" | 过度设计——这是 Pilot，不是正式产品 |
| 改数据库或 API | 与可用性无关 |

## 6. Flow Guidance Improvements

### Before (Pilot #1)

```
S4 填写 → [保存服务记录] → S5 反馈填写 → [保存反馈]
  → ✓ 反馈已记录
      [返回客户总览]  ←── 店员最常点的
      [创建随访 →]    ←── 容易被忽略
```

**问题：** 两个按钮并列，优先级不明确。店员自然选择第一个或更熟悉的。

### After (Pilot #2)

```
S4 填写 → [保存并继续 →] → S5 反馈填写 → [保存并继续 →]
  → ✓ 反馈已记录
      服务反馈已保存。接下来为这位客户创建随访计划。
      [下一步：创建随访 →]  ←── 蓝色主按钮（全站一致）
      稍后创建随访           ←── 灰色文字链接
      （1.5 秒后自动跳转到随访页）

  → S6 随访页
      创建随访计划，或 [跳过（稍后补录）]
      [创建随访]
```

**改进：**
1. 按钮文案从"保存"改为"保存并继续 →"——店员知道点了会发生什么
2. 成功页主路径视觉优先级 > 次路径
3. 自动跳转降低选择成本——不点就是继续
4. S6 提供明确的跳过选项——不想做随访可以跳过，但不会因为"没看到按钮"而漏掉

## 7. Training Adjustments for Pilot #2

### Briefing 脚本增加 3 句话

**在介绍 6 步流程时增加：**

> "第 4 步到第 6 步是连着的——保存服务后会自动进入反馈，保存反馈后会自动进入随访。你不需要手动跳转，系统会带你走。如果某个客户不需要随访，点跳过就行。"

**在 S5 保存后口头提醒（仅第一次）：**

> "好了，反馈保存了。系统会自动跳到随访页——接下来为这位客户创建一个随访计划。告诉我你想什么时候、用什么方式联系他？"

**在 S6 完成后强调：**

> "完成了！现在回到总览页——你看，服务记录有了，反馈有了，随访也有了。这就是完整的服务闭环。"

## 8. Validation Focus for Pilot #2

### 必须验证的假设

| # | 假设 | 验证方式 |
|---|------|---------|
| H1 | "保存并继续 →" 按钮能减少 S4 犹豫 | 观察店员在 S4 的停留时间 |
| H2 | 自动跳转到 S6 能减少漏掉率 | 统计 S6 完成率（对比 Pilot #1） |
| H3 | "下一步：创建随访 →" 作为主按钮更明确 | 观察店员是否仍点"稍后创建" |
| H4 | 中文按钮文案被店员理解 | 询问店员每个按钮的意思 |
| H5 | 店员第二个客户比第一个快 50% | 计时对比 |

### 不验证的内容

- 手机端操作（设备兼容性——如果店里有手机可以在闲时测试）
- 多店员同时操作（单店 Pilot）
- 压力测试（Pilot 客户量不够）

## 9. Exit Criteria for Pilot #2

### Pilot #2 成功标准

- [ ] S6 完成率 ≥ 80%（即 10 个客户中至少 8 个创建了随访）
- [ ] S4 保存动作无人犹豫超过 5 秒
- [ ] 店员独立完成第二个客户（无需口头提醒）
- [ ] 店员能解释"为什么需要创建随访"
- [ ] 无新增 P0/P1 bug

### Pilot #2 失败标准

- [ ] S6 完成率 < 50%
- [ ] S4 保存仍有店员明显犹豫
- [ ] 店员多次口头询问"接下来做什么"
- [ ] 新增 P0 bug

### 如果 Pilot #2 通过

→ S5→S6 自动跳转 + 按钮文案可作为正式 UI 方案保留
→ 进入 Sprint-4 规划（AI 功能）
→ 考虑第三次试运行（多天连续运行）

### 如果 Pilot #2 未通过

→ 回退自动跳转，改为 S5 成功后直接显示 S6（去掉中间成功页）
→ 增加 S6 强制步骤（仅跳过按钮移到右上角极小）
→ Pilot #3 聚焦 S6 完成率
