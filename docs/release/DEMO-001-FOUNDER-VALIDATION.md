## 5. Bugs Found

### B-001: Platform DB Unavailable — No Graceful Degradation

| Attribute | Value |
|-----------|-------|
| **Severity** | **Critical** |
| **Steps** | Start Platform API without PostgreSQL → call `POST /api/identities/` |
| **Expected** | 503 Service Unavailable with message "Database not available" |
| **Actual** | 500 Internal Server Error with raw SQLAlchemy traceback |
| **Root Cause** | Platform DB engine throws `ConnectionRefusedError` which FastAPI converts to 500. No try/except at the endpoint level. |
| **Fix** | Add a FastAPI dependency or middleware that checks DB health before allowing write operations. Return 503 instead of 500. Or: require PostgreSQL to be running before starting the service (startup health gate). |

### B-002: API Version Mismatch with VERSION.md

| Attribute | Value |
|-----------|-------|
| **Severity** | Low |
| **Steps** | `GET /health` → returns version "0.1.0" |
| **Expected** | "0.2.0" matching VERSION.md |
| **Actual** | "0.1.0" — hardcoded in `platform/config.py:25` |
| **Fix** | Update `APP_VERSION` in `PlatformSettings` to "0.2.0". |

### B-003: /health/db Returns 200 When DB Is Down

| Attribute | Value |
|-----------|-------|
| **Severity** | Low |
| **Steps** | `GET /health/db` when PostgreSQL is unreachable |
| **Expected** | HTTP 503 |
| **Actual** | HTTP 200 with JSON `{"platform_db": "error", "detail": "..."}` |
| **Root Cause** | `check_db_connection()` catches the exception and returns a dict. The endpoint returns 200 regardless. |
| **Fix** | Raise `HTTPException(status_code=503)` when `platform_db` status is "error". |
