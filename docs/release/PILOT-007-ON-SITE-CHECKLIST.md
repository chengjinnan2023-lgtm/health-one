## Pre-Pilot Checklist (Morning, Before Store Opens)

### Server & System

- [ ] Store computer powered on, connected to network
- [ ] Open browser → go to `http://<store-server-ip>`
- [ ] Health One login page is visible
- [ ] Test login with staff credentials → success

### Materials Ready

- [ ] Printed PILOT-006 Runbook (1 copy at workstation)
- [ ] Printed Quick Reference Card (1 copy at workstation)
- [ ] Printed Issue Log (PILOT-007-ISSUE-LOG.md) — blank, ready to fill
- [ ] Printed Post-Run Review template (PILOT-007-POST-RUN-REVIEW.md)
- [ ] Pen and paper for manual notes
- [ ] Phone/camera for screenshots if needed

### Staff Ready

- [ ] Staff has completed training (PILOT-001 §5)
- [ ] Staff knows their username and password
- [ ] Staff understands the 6-step loop
- [ ] Staff has practiced at least once independently
- [ ] Staff knows: "If stuck, serve customer first, record later"

### Customer Consent Script Ready

Staff should say:

> "We're testing a new health management system today. With your permission, I'll record your service and feedback in the system — just your name, health concern, and how you felt after service. Your information stays private. You can say no at any time."

---

## During Pilot (Per Customer)

### Before Service

- [ ] Greet customer → explain pilot (consent script)
- [ ] If consent given: proceed to Step 1
- [ ] If consent declined: serve as usual, do NOT record in system

### Service Loop (Follow Runbook §4)

- [ ] Step 1: Search or create customer
- [ ] Step 2: Activate (if new) → view summary
- [ ] Step 3: Record health concern (category + description)
- [ ] Step 4: Record service (type + detail)
- [ ] Step 5: Capture feedback (feeling + satisfaction + return)
- [ ] Step 6: Create follow-up (method + date)

### Per-Customer Verification

- [ ] All 6 steps completed? ___ Yes ___ No (which step skipped?)
- [ ] Timeline shows today's events? ___ Yes ___ No
- [ ] Any errors? ___ Yes ___ No (log in Issue Log)
- [ ] Time taken for full loop: _____ minutes

## End-of-Pilot Checklist (End of Day)

### Data Verification

- [ ] Count total customers served: _____
- [ ] For each customer: open their Summary page → Timeline shows today's events
- [ ] For each customer: Sessions tab shows today's service
- [ ] For each customer: Plans tab shows follow-up status
- [ ] Any missing data? ___ Yes ___ No (describe: _______________)

### System Check

- [ ] System still accessible (no crash during day)
- [ ] Login still works
- [ ] Customer Search still works
- [ ] New customer creation still works

### Backup

- [ ] Store manager executes database backup
- [ ] Backup file location: _______________
- [ ] Backup file size: _____ KB

### Documentation

- [ ] Issue Log completed (all problems documented)
- [ ] Staff feedback collected (see Post-Run Review)
